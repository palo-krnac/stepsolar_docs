# ADS1115 Wiring & Calibration

## Wiring

```
ADS1115
┌──────────────┐
│ VDD ─── 3.3V │
│ GND ─── GND  │
│ SCL ─── GPIO9│
│ SDA ─── GPIO8│
│ ADDR ── GND  │  → I2C address 0x48
│              │
│ AIN0 ──────────── Current sensor output
│ AIN1 ──────────── Voltage divider mid-point
│ AIN2 ──────────── KTY81/210 mid-point
│ AIN3 ──────────── (free)
└──────────────┘
```

## Gain Setting

All channels use **GAIN_ONE (±4.096V)**, LSB = 0.125 mV.

```cpp
_ads.setGain(GAIN_ONE);
```

## Current Measurement (AIN0)

Designed for **ACS712-20A** (100 mV/A, output centred at Vcc/2):

| Parameter | Value |
|-----------|-------|
| Sensitivity | 100 mV/A |
| Zero-current output | 1.65V (at 3.3V supply) |
| Max measurable | ±16.5A |
| Firmware limit | 10.00A (overcurrent at 10A) |

Calibration in `config.h`:
```cpp
#define CURRENT_SENSITIVITY  0.1f   // V/A — change for your sensor
```

And in `measurement.h`:
```cpp
float vMid  = 1.65f;   // Adjust if your sensor midpoint differs
```

## Voltage Measurement (AIN1)

Resistive divider scales PV voltage to ADS1115 range:

```
Vmax_panel = 4.096 × (R1 + R2) / R2
           = 4.096 × (470000 + 5600) / 5600
           = 347.5 V
```

Calibration offset (`kalibV`) adjustable via web UI in 0.1% steps.

## Temperature Measurement (AIN2)

KTY81/210 silicon temperature sensor in a voltage divider with 2.2 kΩ:

```
3.3V ──[2200Ω]──┬── AIN2
                │
             [KTY81/210]
                │
               GND
```

Disconnected sensor detection: if AIN2 voltage > 98% of supply, temperature is reported as `--` (code `-99`).

!!! tip "Calibration"
    Use the **V calibration** and **A calibration** fields in the Settings tab to apply percentage offsets (range ±5.0%) without recompiling firmware.
